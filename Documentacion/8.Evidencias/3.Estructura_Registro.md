# Estructura de Registro de Evidencias

## 1. Descripción General

Este documento define la estructura estandarizada para el registro y documentación de evidencias en el proyecto IAEDU1. Proporciona plantillas, formatos y procedimientos para asegurar una documentación consistente y completa de todas las actividades y entregables del proyecto.

## 2. Plantillas de Registro

### 2.1 Evidencia de Desarrollo
```markdown
# Registro de Desarrollo

## Información General
- **ID:** DEV-[YYYYMMDD]-[XXX]
- **Tipo:** [Feature|Bug|Enhancement]
- **Módulo:** [Nombre del módulo]
- **Desarrollador:** [Nombre]
- **Fecha:** YYYY-MM-DD

## Descripción
[Descripción detallada del desarrollo]

## Cambios Realizados
- Archivos modificados:
  1. [archivo1]
  2. [archivo2]
- Funcionalidades:
  1. [funcionalidad1]
  2. [funcionalidad2]

## Pruebas
- [ ] Pruebas unitarias
- [ ] Pruebas de integración
- [ ] Pruebas de usuario

## Resultados
[Descripción de los resultados]

## Documentación Relacionada
- [Enlaces a documentos]
- [Referencias]
```

### 2.2 Evidencia de Testing
```markdown
# Registro de Pruebas

## Información General
- **ID:** TEST-[YYYYMMDD]-[XXX]
- **Tipo:** [Unit|Integration|User]
- **Módulo:** [Nombre del módulo]
- **Tester:** [Nombre]
- **Fecha:** YYYY-MM-DD

## Casos de Prueba
1. Caso #1
   - Input:
   - Output esperado:
   - Resultado:
   - Estado: [Pass|Fail]

2. Caso #2
   - Input:
   - Output esperado:
   - Resultado:
   - Estado: [Pass|Fail]

## Defectos Encontrados
1. Defecto #1
   - Descripción:
   - Severidad:
   - Pasos para reproducir:
   - Estado:

## Métricas
- Total casos: XX
- Pasados: XX

```

### 3.1 Organización
```

├── desarrollo/
│   ├── YYYY-MM/
│   │   ├── DEV-20240320-001/
│   │   │   ├── registro.md
│   │   │   ├── capturas/
│   │   │   └── codigo/
│   │   └── DEV-20240320-002/
│   └── YYYY-MM/
├── pruebas/
│   ├── YYYY-MM/
│   │   ├── TEST-20240320-001/
│   │   │   ├── registro.md
│   │   │   ├── resultados/
│   │   │   └── reportes/
│   │   └── TEST-20240320-002/
│   └── YYYY-MM/
└── documentacion/
    ├── YYYY-MM/

    │   │   ├── archivos/
    │   │   └── recursos/
    │   └── DOC-20240320-002/
    └── YYYY-MM/
```

## 4. Proceso de Registro

### 4.1 Flujo de Trabajo
1. Crear directorio
2. Inicializar plantilla
3. Documentar actividad
4. Adjuntar recursos
5. Revisar contenido
6. Actualizar índices

### 4.2 Validación
- Completitud
- Consistencia
- Enlaces válidos
- Recursos adjuntos
- Formato correcto

## 5. Metadatos

### 5.1 Información Básica
```yaml
---
id: [ID único]

modulo: [Módulo relacionado]
autor: [Nombre del autor]
fecha: YYYY-MM-DD
```
